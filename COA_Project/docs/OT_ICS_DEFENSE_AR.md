# OT / ICS والدفاع عن العالم المادي

## ما الفرق بين IT و OT و ICS؟

| المصطلح | المعنى |
|---------|--------|
| **IT** | الأنظمة التي تتعامل مع **البيانات**: خوادم، بريد، قواعد بيانات، مكاتب. |
| **OT** | الأنظمة التي تتحكم في **العالم المادي**: محركات، صمامات، خطوط إنتاج، شبكات كهرباء، محطات مياه. |
| **ICS** | المظلة لأنظمة **التحكم الصناعي**، وتشمل SCADA، PLC، DCS، HMI، RTU، Historian، إلخ. |

## لماذا يهم الهاكاثون الدفاعي؟

- **مشهد تهديد حقيقي:** Stuxnet، BlackEnergy، **TRITON/TRISIS** (2017 — استهداف أنظمة سلامة في منشأة بتروكيماويات في المملكة)، Industroyer2، Pipedream/Incontroller.
- **سياق محلي:** أرامكو، سابك، الكهرباء، التحلية، الموانئ، **SAMI** — تعتمد على OT/ICS.
- **فجوة سوق:** كثير من الحلول تركز على IT؛ حلول OT المتخصصة غالباً أجنبية وباهظة وقد تعتمد على سحابة.

## لماذا OT مختلف عن IT؟

| الجانب | IT | OT |
|--------|----|----|
| الأولوية | السرّية غالباً | **التوفّر** والسلامة أولاً |
| دورة الحياة | سنوات قليلة | **15–30 سنة** أحياناً |
| التحديثات | متكررة | **نادرة** (التوقف مكلفاً) |
| البروتوكولات | TCP/IP، HTTPS | Modbus، DNP3، S7، OPC، Profinet، … |
| الفحص النشط | شائع | **غير مقبول** افتراضياً — قد يعطّل إنتاجاً |

## مبدأ C.O.A في OT

**Passive-by-Default في بيئات OT**

- التحليل الافتراضي يعتمد على **مراقبة سلبية**: جدول الاتصالات والعمليات الموجود لدى نظام التشغيل (مثل `psutil`) — **بدون** فحص نشط لبروتوكولات صناعية.
- أي إجراء **حظر / عزل / إيقاف** يتطلب **موافقة مشغّل OT**، نافذة صيانة، تقييم أثر إنتاجي، وخطة تراجع — هذا **تصميم وثائقي**؛ التنفيذ الفعلي يبقى عند المشغّل.

## ما الذي يضيفه المشروع تقنياً؟

| المكوّن | الوصف |
|---------|--------|
| **ICS Protocol Analyzer** | `ics_analyzer/` — استنتاج بروتوكولات شائعة من **المنافذ** في اتصالات الشبكة المعروضة (سلبي). |
| **Passive asset sketch** | تجميع عنوان IP + منافذ ICS + عمليات مرتبطة — **ليس** مخزوناً كاملاً بدون PCAP/مرآة منفذ. |
| **ICS Specialist (Agent #6)** | تقييم أثر تشغيلي/سلامة مبسّط ونص توصيات «آمنة للإنتاج». |
| **سيناريوهات OT** | ملفات YAML في `defense_playbooks_ot/` (Stuxnet-like، TRITON-like، …) كفرضيات تُقيَّم من المؤشرات المتاحة. |
| **ICS-CERT / CVE** | مجلد `ics_vulnerabilities/` مع README لربط **حقيقي** لاحقاً ببيانات CISA — **لا** نُضمّن CVE وهمية. |
| **لوحة OT في React** | مسار `#/ot-dashboard` + تبويب **OT/ICS** في لوحة الفحص. |
