# واجهة React — الإضافات وصفحات الاختبار

هذا الملف يشرح **كل إضافة** تم ربطها بمسارات منفصلة في الواجهة، **لماذا** وُجدت، وكيف تشغّل النظام بالكامل للاختبار.

## أوامر تشغيل النظام (كامل)

من جذر المشروع البرمجي `COA_Project/` (المجلد الذي فيه `web_api.py` ومجلد `web/`).

### الطرفية 1 — خادم الـ API (Python)

```bash
cd /Users/ahmed/Desktop/COA/COA_Project
python web_api.py
```

- لتفعيل **تشغيل pytest من الواجهة** (`#/dev-tests`): ابدأ الخادم بـ `COA_ALLOW_DEV_TESTS=1 python web_api.py` (لا تستخدم ذلك على خوادم إنتاج).

- يستمع عادة على: `http://127.0.0.1:5050`
- مسار الفحص: `POST http://127.0.0.1:5050/api/scan`

### الطرفية 2 — واجهة React (Vite)

```bash
cd /Users/ahmed/Desktop/COA/COA_Project/web
npm install
npm run dev
```

- المتصفح: `http://localhost:5173` (أو ما يعرضه Vite)
- التوجيه يستخدم **HashRouter**؛ المسارات تظهر كـ `#/dashboard` إلخ.

### خطوة الاختبار الموحّدة

1. افتح الرابط أعلاه.
2. انتقل إلى **فحص / لوحة** (`#/dashboard`) واضغط **Start scan** مرة واحدة على الأقل.
3. بعد نجاح الفحص، تُخزَّن نتائج جزء من الوكلاء في **`sessionStorage`** تحت المفتاح `coa_last_scan_extras` (وبعضها في `coa_defense_context`).
4. افتح أي صفحة من الجدول أدناه؛ كل صفحة تعرض **ميزة واحدة** بشكل أوضح للمراجعة.

### (اختياري) واجهة سطح المكتب أو الـ CLI

```bash
cd /Users/ahmed/Desktop/COA/COA_Project
python gui.py
```

أو:

```bash
python main.py
```

الواجهة الحديثة للعرض والاختبار السريع هي **React + web_api** كما فوق.

## شرح كل إضافة ولماذا وُجدت

### 1) لوحة الفحص `#/dashboard`

- **ماذا:** نقطة تشغيل الفحص، عرض التهديدات والعمليات والاتصالات، وتبويب **OT/ICS** المختصر مع رابط للوحة OT الكاملة.
- **لماذا:** تبقى «الوحدة المركزية» التي تغذي بقية الصفحات بالبيانات عبر `sessionStorage` بعد `POST /api/scan`.

### 2) صفحة سياق دفاعي `#/defense-context` (الوكيل #5)

- **ماذا:** تفاصيل `defense_context` من آخر فحص: إسناد، ملفات APT مرتبة، playbooks، نصوص استراتيجية، وقائمة موقف دفاعي؛ مع JSON لـ `mitre_heatmap` للتحقق السريع من البيانات الخام.
- **لماذا:** فصل العرض عن لوحة الفحص يتيح للمختبر أو المقدّم رؤية **طبقة الاستراتيجية الإقليمية / APT** دون ضجيج بقية الجداول.

### 3) صفحة MITRE العميق `#/mitre-deep`

- **ماذا:** حزمة `mitre_deep`: مراحل تكتيكية مرتّبة، تلميحات فجوات، ملاحظة ICS، تقرير ASCII طويل، وزر لتنزيل `navigator_layer` من الجلسة (JSON).
- **لماذا:** «العمق» (D3FEND، تسلسل تكتيكي، Navigator) مختلف عن **الشبكة الحرارية**؛ فصلهما يوضح قيمة المشروع في **تقارير قابلة للتصدير** وليس فقط واجهة ملوّنة.

### 4) صفحة خريطة MITRE `#/mitre-heatmap`

- **ماذا:** شبكة تقنيات مع درجات حرارة (heat) وفلاتر عرض؛ مع رابط إلى صفحة العمق لتفاصيل إضافية وتصدير Navigator.
- **لماذا:** مناسبة لـ **عرض live** أمام الجمهور (قراءة سريعة لتغطية ATT&CK).

### 5) صفحة OT/ICS `#/ot-dashboard` (+ تبويب في لوحة الفحص)

- **ماذا:** تحليل **سلبي** لمنافذ وبروتوكولات صناعية شائعة من جدول الاتصالات، سيناريوهات `defense_playbooks_ot`، درجة استمرارية إنتاج تجريبية، وتقرير **ICS Specialist (الوكيل #6)**.
- **لماذا:** إبراز تمييز المشروع عن أدوات IT فقط؛ والالتزام بمبدأ **Passive-by-default** في سياق OT (لا فحص نشط من الواجهة).

### جلسة جاهزة عبر الـ API (اختياري)

- **`POST /api/demo/seed-session`:** يبني جلسة كاملة الشكل في الخادم و`sessionStorage` إذا استدعيته من أداة خارجية (بدون زر في الواجهة الرئيسية). الفحص الاعتيادي للمستخدم هو **`POST /api/scan`** من لوحة الأداء ويجمع من المضيف الحقيقي.
- **`GET /api/demo/ot-ics-fixture`:** حزمة OT من [`fixtures/demo_ot_ics.json`](../fixtures/demo_ot_ics.json) للاختبار اليدوي عند الحاجة.

### 6) صفحة pytest `#/dev-tests`

- **ماذا:** أزرار تشغيل `python -m pytest` على مجلد `tests/` (أو مجموعة «سريعة» من ملفات الاختبار الحالية) وعرض المخرجات في الصفحة.
- **لماذا:** تسهيل **التحقق السريع** أثناء العرض أو الهاكاثون دون التبديل للطرفية؛ مع **تعطيل افتراضي** حتى لا يصبح تشغيل أوامر على الخادم ثغرة أمنية في النشر.

## ملاحظات تقنية مختصرة

- **مصدر البيانات للصفحات المنفصلة:** آخر فحص ناجح من React؛ المفتاح الرئيسي `coa_last_scan_extras` يحتوي عادةً: `defense_context`, `mitre_deep`, `ot_ics`. بعد كل فحص ناجح يُطلق الحدث `coa-scan-complete` في النافذة نفسها، ويُحدَّث `sessionStorage` فيُطلق `storage` في النوافذ الأخرى — لتُحدَّث الصفحات المفتوحة دون إعادة تحميل يدوية للمتصفح.
- **بدون فحص:** الصفحات تعرض رسالة توجيه للعودة إلى `#/dashboard`.
- **تصدير Navigator من الخادم (وليس من الجلسة فقط):** بعد الفحص، يمكن استدعاء `GET http://127.0.0.1:5050/api/reports/mitre-navigator.json` إن كان الخادم يحتفظ بآخر فحص في الذاكرة.
- **تشغيل الاختبارات من الـ API:** `GET /api/dev/tests-enabled` — `POST /api/dev/run-tests` مع جسم JSON اختياري `{"scope":"all"}` أو `"quick"` (يُفعّل فقط مع `COA_ALLOW_DEV_TESTS=1`).
