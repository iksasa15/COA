# الوكلاء والطبقات — نسخة للتقديم (C.O.A)

ملخص يمكن قراءته في دقيقتين أو تحويله لشرائح. للتفاصيل التقنية والملفات: [AGENTS_REFERENCE.md](AGENTS_REFERENCE.md) ومجلد [wokala/](wokala/).

---

## جملة واحدة للمقدّم

**C.O.A** يجمع بين **تحليل أمني حتمي محلي** (قواعد، درجات تهديد، YARA، اختياري VirusTotal) وبين **مجلس ثلاثي من وكلاء CrewAI على Ollama** يُفعَّل عند الحاجة لتلخيص وتفسير بلغة طبيعية، ثم **تقارير حوادث** و**سياق دفاعي** و**OT/ICS** دون إرسال بيانات حساسة لسحابة افتراضية.

---

## طبقتان مهمّتان (لا تخلطهما في الشرح)

| الطبقة | المعنى للجمهور | يحتاج Ollama؟ |
|--------|----------------|---------------|
| **المحرك الحتمي** | «العقل القانوني»: يقرأ بيانات الجهاز ويُنتج قائمة تهديدات موحّدة تغذي الحلول والتقارير. | لا |
| **مجلس الوكلاء 1–3** | «المحلل اللغوي»: يعيد صياغة وتفسير ما جمعه المحرك (تقرير نصي للمجلس). | نعم (CrewAI + Ollama) |

الفحص الكامل يعمل **بدون** المجلس؛ المجلس يضيف طبقة تفسيرية عند تفعيله (`--council` أو من الواجهة بـ `use_council`).

---

## من يفعل ماذا؟ (جدول للشرائح)

| الاسم المعروض | الدور باختصار | تقنية |
|---------------|----------------|--------|
| **جمع البيانات** (`data_collector`) | يجمع معلومات النظام الفعلية (عمليات، شبكة، …). | Python — ليس CrewAI |
| **Threat Analyzer** | يحوّل المدخلات إلى تهديدات مصنّفة ودرجات؛ قلب المسار. | Python حتمي |
| **The Eye** | يُنظّم ويلخّص صورة النظام بلغة طبيعية للمجلس. | CrewAI + Ollama |
| **The Brain** | يفسّر ويوسّع من زاوية «صيد التهديدات». | CrewAI + Ollama |
| **The Strategist** | يقدّم توصيات استراتيجية نصية في سياق المجلس. | CrewAI + Ollama |
| **Incident Reporter** | يولّد تقارير حوادث (Markdown وغيرها) مع ربط MITRE حيث ينطبق. | Python حتمي |
| **Defense Context** | يثرّي التهديدات بسياق من ملفات YAML محلية (سياسات/سياق دفاعي). | Python حتمي |
| **ICS Specialist** | طبقة تخصص لبيئات OT/ICS بعد تحليل المسار ذي الصلة. | Python حتمي |
| **Solution Engine** | يبني حلولاً قابلة للمراجعة والموافقة البشرية من قائمة التهديدات. | ليس وكيل CrewAI |

**YARA / VirusTotal:** إضافات اختيارية على مسار التحليل، وليست «وكلاء» بمعنى CrewAI.

---

## تسلسل يمكن رسمه على السبورة (بسيط)

1. جمع → 2. تحليل تهديدات (+ YARA/VT اختياري) → 3. إثراء (سياق دفاعي + OT إن وُجد) → 4. (اختياري) مجلس Ollama للتقرير النصي → 5. محرك الحلول + موافقة → 6. تقرير الحادث.

---

## ماذا تقول إن سُئلت عن «الذكاء»؟

- **الذكاء المحلي:** Ollama على الجهاز أو الشبكة الداخلية؛ لا إلزام بإرسال بيانات حساسة لنموذج سحابي عام.
- **الفرق بين «كشف» و«تفسير»:** الكشف والأولويات من المحرك الحتمي؛ المجلس يساعد المحلل البشري على **القراءة والتوثيق** بسرعة.

## أوامر سريعة للعرض الحي (من `COA_Project/`)

```bash
source venv/bin/activate
python main.py                    # فحص بدون مجلس LLM
python main.py --council          # يشغّل مسار CrewAI بعد الجاهزية
python -m agents.council          # اختبار اتصال Ollama والمجلس
```

الواجهة: تشغيل `web_api.py` + `npm run dev` كما في [run.md](../../run.md) في جذر المستودع.
